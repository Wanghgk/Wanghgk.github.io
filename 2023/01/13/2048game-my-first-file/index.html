2048game, my first file

Posted on 2023-01-13

点此进入游戏

本文所介绍项目点此链接查看代码

组件介绍——html和css属性

1、背景

纯色背景，display:grid，左右两列80%，20%

2、功能区

标题区：2048

游戏区：4*4的方块，a，b，c，d命名行，1，2，3，4命名列。

3、菜单

从上到下依次是“开始游戏”、“重新开始”、“上”“下”“左”“右”三组按键

4、结束标志

若16个方块均被填满，则游戏结束，出现棕色Wasted

若出现2048则游戏胜利，出现金色Win

JavaScript部分

概述

总的思路是用一个二维数组储存16个方块的id，当上下左右四个按键被点击时，执行移动命令并刷新小方块颜色，之后检查16个小方块是否都被填满或是否存在2048，若满足条件则游戏结束。

清除函数

清除指定方块的数字并将背景颜色设置为初始颜色

染色函数

将指定快的背景颜色依据块的html内容设定成指定颜色（本文设定最大值为2048）

游戏胜利

由check函数调用。

显示“重新开始”按钮，显示win效果。

隐藏上下左右按钮，隐藏“开始游戏”按钮（本来就是隐藏的）。

游戏失败

由check函数调用。

显示“重新开始”按钮，显示wasted效果。

隐藏上下左右按钮，隐藏“开始游戏按钮”（本来就是隐藏的）。

检查确定是否结束

遍历二维数组，若存在2048则游戏胜利，若不存在且16个格均填满则游戏失败结束。

实现：遍历数组，若方块html值为2048则游戏胜利，调用游戏胜利的函数，否则，若html值不为空则cnt++（cnt从0开始计），若游戏没有胜利则判断cnt值，若cnt值等于16（每个格子都有数字且没有2048），则游戏失败结束，调用游戏结束的函数。

生成随机数字

第一步，选出空格子：设两个一维数组，分别用来储存html值为空的块的i，j值（在二维数组中的坐标）。

第二步，在随机空格子生成随机数字：根据一维数组的长度生成一个数字，作为下标选中两个一维数组中的一组值，再用该值作为二维数组下标选中一个空格子，将这个格子的html值随机设为2或4并调用染色函数。

开始游戏

当“游戏开始”按钮被点击时，调用随机生成函数，隐藏“游戏开始”按钮，显示“重新开始”按钮。

重新开始

当“重新开始”按钮被点击，隐藏胜利或失败标志，，隐藏“重新开始”按钮。

显示“开始游戏”按钮，显示上下左右按钮。

移动——以向上移为例

移动是本项目的核心要素，这里我们将尽量描述函数的思路，具体还请参考具体函数。

首先，是如何实现移动。以向上移为例，我们选中第二行第一个方块b1，再选中它正上方相邻的方块——也就是第一行第一个方块a1。如果a1的数字为空或者a1的数字和b1一致，那么我们将b1的数字加到a1中，然后将b1的值设为零（事实上具体实现的时候并不能这么简单的加上，而是要根据a1的两种情况分开实现，具体请参考具体函数内容）。我们从上到下遍历储存id的二维数组就能实现让每个元素移动一次。

然后是让“同一个”块移动到头（也就是移动到不该再移动）。我们用一个循环来实现这个目标：还是以上移为例，我们将轮换的内容放入循环，在最后加上一个行数减一的命令rowx--。当romx>0时，循环继续。另外，我们设定一个moveflag=true，当上下的一对元素中上面的元素不为空，我们就将moveflag=false，这样就可以更早地结束循环。

再然后是限制移动的距离。目前我们的移动函数会在上下都有数字且数字不相等的情况下会停止移动，但在某些情况下可能会连续合成，我们尽力避免这种情况。我们把连续合成简单分成两种情形，一种是两个2先移动合成4，然后另一个4再移动，两个4再合成8；另一种是一个4先移动到底，两个2再移动合成一个4，而这个4再移动合成8。对于前者，我们设置一个对应16个格子的二维数组，将16个元素都设为0，若某个格子经过了一次合成，那么这个格子的对应位置的数字由0改为1，我们在移动的条件中加入了mover[romx-1][romy]===0的要求，那么这个元素在这一次遍历中就不会再被合并了；然而我们可以发现，这种方法对于后者的情况是无效的（因为移动到底的4并不会被修改对应的mover数组的值），但这种情况并不会出现，因为我们在上一段中使用的moveflag在这时已经将循环结束了，这两个数字并不会合并。

之后是在随机空格生成一个随机的2或4。我们声明一个未知量let succes=false，如果在接下来的遍历数组过程中有任何一个方块成功移动我们就将succes的值设为真。在遍历数组结束之后，若succes的值为真，我们调用random_span函数。

最后我们调用check函数，查看游戏是否结束。
